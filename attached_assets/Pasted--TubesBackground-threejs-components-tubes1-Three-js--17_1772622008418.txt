Компонент `TubesBackground` использует библиотеку `threejs-components/tubes1` (Three.js анимация неоновых трубок). По умолчанию эта библиотека делает так, что трубки следуют за курсором мыши или пальцем — это мешает нормальному скроллу и взаимодействию со страницей. Мы отключили это поведение и превратили анимацию в чисто декоративный фон, который не реагирует ни на мышь, ни на касания.

### Как это работает — 3 уровня защиты:

#### Уровень 1: CSS `pointer-events: none` (блокировка на уровне браузера)

На всех контейнерах и на самом iframe стоит `pointer-events: none`. Это значит, что браузер вообще не передаёт никакие нажатия, движения мыши или касания пальцем внутрь этих элементов. Все клики и свайпы «проваливаются» сквозь фон и попадают на контент страницы, который лежит поверх.

```tsx
// На iframe (inline стиль):
iframe.style.cssText = '...pointer-events:none;';

// На контейнерах (Tailwind классы):
<div className="... pointer-events-none">     // внешний контейнер
  <div className="... pointer-events-none">   // контейнер iframe
  <div className="... pointer-events-none">   // градиентный оверлей
  <div className="... pointer-events-none">   // обёртка children
```

#### Уровень 2: JavaScript-блокировка всех событий ввода внутри iframe

Даже если какое-то событие каким-то образом проникнет внутрь iframe, там стоит жёсткий перехват. Блокируются ВСЕ типы событий ввода на трёх уровнях (document, window, canvas):

```javascript
const block = [
  'mousemove','mousedown','mouseup',
  'touchstart','touchmove','touchend',
  'pointerdown','pointermove','pointerup',
  'click'
];
block.forEach(e => {
  document.addEventListener(e, ev => {
    ev.stopImmediatePropagation();  // остановить распространение
    ev.preventDefault();            // отменить действие
  }, { capture: true, passive: false });
  // То же самое на window и canvas
});
```

Флаг `capture: true` — перехват на фазе захвата (самый ранний момент), событие не дойдёт до библиотеки.
Флаг `passive: false` — разрешает вызывать `preventDefault()`.

#### Уровень 3: Конфигурация библиотеки + заморозка координат мыши

При создании экземпляра `TubesCursor` передаются параметры, отключающие реакцию на мышь:

```javascript
const app = TubesCursor(canvas, {
  tubes: { count: 8, radius: 0.08, colors: [...], lights: {...} },
  mouse: { disabled: true, lerp: 0 },   // ← отключить отслеживание мыши
  cursor: { enabled: false }             // ← отключить кастомный курсор
});
```

После создания объекта координаты мыши «замораживаются» — свойства перезаписываются через `Object.defineProperty`, чтобы они всегда возвращали `0` и не могли быть изменены:

```javascript
if (app && app.mouse) {
  Object.defineProperty(app.mouse, 'x', {
    get: () => 0, set: () => {}, configurable: true
  });
  Object.defineProperty(app.mouse, 'y', {
    get: () => 0, set: () => {}, configurable: true
  });
  // То же для lerpX и lerpY
}
```

Даже если библиотека внутри попытается прочитать позицию курсора, она всегда получит `(0, 0)`. Трубки остаются в центре и анимируются автономно — красивый живой фон без реакции на ввод.

### Почему iframe?

Анимация живёт внутри изолированного iframe, а не напрямую в DOM страницы. Это даёт:
- **Изоляцию** — Three.js контекст и все обработчики событий полностью изолированы от основного приложения
- **Производительность** — рендеринг canvas в отдельном контексте браузера
- **Безопасность** — никакие скрипты библиотеки не могут повлиять на основную страницу

### Итог

Три уровня защиты работают вместе:
1. CSS не пускает события внутрь (pointer-events: none)
2. JavaScript блокирует любые события, если они всё же попадут (stopImmediatePropagation + preventDefault)
3. Библиотека настроена не реагировать на мышь, а координаты заморожены на (0,0)

Результат: трубки красиво анимируются сами по себе как чисто декоративный фон, пользователь свободно скроллит и нажимает на контент поверх.
